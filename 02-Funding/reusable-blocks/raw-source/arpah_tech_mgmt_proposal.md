# **[Technical Plan]{.smallcaps}** 

1.  Present a credible plan to achieve the project's goal. High-risk,
    high-reward projects are welcome.

2.  Outline and address the technical challenges inherent in the
    approach and possible solutions for overcoming potential problems.

3.  Provide appropriate measurable milestones (quantitative if possible)
    at intermediate stages of the project to demonstrate progress;
    milestones should be clearly articulated and defined in time
    relative to the start of the project.

4.  Provide a schedule showing milestones, tasks, and the
    interrelationships among tasks. The task structure must be
    consistent with that in the Task Description Document (TDD).
