# Connecting Cowork, Claude Code, and Claude Design, in Plain Words

**Reading time: about 1 minute.**

> **Full version:** `../COWORK-CODE-DESIGN-INTEGRATION-STANDARD_2026-07-11.md`.

> **If you only read one thing:** the three tools live in one desktop app on one account. Keep it open, set up connectors once, and they can all reach each other and your files. A few built-in bridges do the rest.

## The simple setup
- **One app, one login, always open.** That app is what lets Cowork touch your laptop files and browser.
- **Connectors once:** add them one time; Chat, Cowork, and Code all get them.
- **Code <-> Design:** built-in commands (`/design-sync`, a Design connector, and a "hand off to Claude Code" button) connect them directly.
- **Anything with no direct connection:** Claude drives it through the Chrome extension (reads and clicks the page).
- **Full computer access:** the host terminal tool already gives Claude real access to your files and git.

## Good to know
- The "folder not trusted" popups are now pre-approved, so launches just work.
- Auto-running Claude Code in the background needs a one-time sign-in (that was the 401 earlier).
- Since this is becoming company policy, a quick lawyer check is worth it.
