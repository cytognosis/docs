### Known Limitations

- Structural conformance does not prove clinical validity, therapeutic quality, model truthfulness, or semantic quality.
- The semantic-quality harness and synthetic fixtures prove only the evaluation workflow shape; they do not replace qualified expert review or governed datasets.
- The regulated-profile review packet proves only the review workflow shape and evidence map; it does not replace qualified external profile review, regulatory clearance, clinical approval, or deployment-specific privacy/legal review.
- Schema scaffolding plus the migrated gRPC and HTTP/JSON hot paths does not imply full CAP v1 runtime migration.
- Local demos and fixtures do not imply live multi-organization interoperability.
- The live MCP/A2A substrate scaffold is local and deterministic. It proves CAP enforcement before local handlers run, not external MCP server behavior, A2A directory compatibility, or multi-organization interoperability.
- The Go third implementation adapter is local fixture validation. It proves independent CAPEnvelope/JCS/signature parsing for a small shared suite, not a production Go runtime or external interoperability certification.
- Lifecycle FSM and profile-inheritance rules are machine-checkable deterministic scaffolds. They do not certify deployed state persistence, production Interrupt Manager coordination, profile-owner change control, or external profile conformance.
- Replay/idempotency protection is currently an in-memory Local PEP scaffold and deterministic test harness; production deployments still need durable per-session replay caches or nonce stores shared across executor replicas.
- Offline policy bundles validate metadata and can verify detached-JWS signed payloads with a supplied key registry. The reference Policy Registry adds signed-bundle distribution, explicit hot update, online revocation refusal, audited emergency override, and per-session pinning, but production deployments still need network service packaging, service authentication, HA replication, monitoring, organization rollout controls, and KMS/HSM signing custody.
- Registry clients now use a SQLite-backed reference service for persistence, live revocation freshness, federation hooks, audit events, and warrant-key lookup. Biscuit warrant verification can require this service for active key lookup and revocation freshness. Capability metadata uses one closed shape across agent, tool, service, human, and policy subjects, but production deployments still need signed distribution, network service packaging, service authentication, HA replication, monitoring, organization rollout controls, deployed revocation operations, and KMS/HSM custody.
- Observability-plane sinks, signed audit operations, Sigstore/Rekor-style transparency bundles, W3C PROV store wiring, and OTel attribute checks are in-process deterministic scaffolds; production deployments still need deployed KMS/HSM custody, external Rekor publication/monitoring, transparency/replication services, deployed OpenTelemetry collector/exporter operations, production PROV graph/document store operations, production access-control integration, recovery, and runtime integration.
- Sidecar bypass checks now include deterministic Android/iOS separately privileged proxy contracts and attested Local PEP registration smoke checks, but production deployments still need native project wiring, platform entitlements, device/instrumented tests, real platform attestation verifiers, and review evidence.
- Benchmark artifacts now quantify local deterministic p50/p95 overhead for direct MCP/tool versus CAP-mediated paths, Local PEP output gating, stream gating, Edge PEP verification, and Android/iOS proxy scaffold paths. They do not measure production model providers, native mobile devices, service meshes, networked registries/PDPs, OS battery APIs, or power-rail/device-profiler battery drain.
- Local NER redaction is deterministic and dependency-free by default, with optional caller-supplied local model spans. It is not a production redaction-quality guarantee and does not allow remote NER before egress; deployments still need model selection, evaluation fixtures, monitoring, and profile-owner review.
- Embedding-only egress uses deterministic dependency-free text/voice vectors by default. It proves the raw-data egress shape and recipient binding, not production embedding quality, semantic privacy, inversion resistance, or model suitability; deployments still need local encoder selection, privacy evaluation fixtures, monitoring, and profile-owner review.
- Policy hot update is explicit: service-backed sessions remain pinned to their existing bundle until `allow_policy_hot_update` or `allow_emergency_policy_override` is enabled. Emergency overrides produce service audit evidence and still require valid signature, digest, and expiry.
- PrivacyBoundary PDP evaluation and the OPA/Cedar adapter parity checks are deterministic and in-process; production deployments still need local and central PDP service boundaries, organization-owned OPA/Cedar runtime integration, Policy Registry distribution, profile ownership/change control, and release-blocking privacy conformance.
- Evidence tamper checks now include service-backed content-addressed dereference, reference attestation metadata, expired backing-content deletion, and deletion audit records that exclude raw evidence; production deployments still need access-policy enforcement, scheduled retention operations, production attestation signing, and network Evidence Registry hardening.
- Edge PEP binding integration uses in-process key registries, policy refs, privacy-boundary mappings, authority-chain payload extraction, payload-ref callbacks, and deterministic service-mesh topology helpers; production deployments still need PDP/Policy Registry integration, live registry authentication, revocation freshness at deployment scale, and external mesh/sidecar rollout.
- InterruptDecision composition, Human Review escalation, and workflow-engine composition are deterministic in-process reference paths; production deployments still need a logical Interrupt Manager, live stream/UI routing, deployed Temporal/LangGraph workers, production human-review workflows, and cross-service conflict coordination.
- Sample runtime labels such as `reference-sample-jws` indicate reference-binding samples; production-hardening separately verifies Ed25519 detached-JWS and DSSE artifacts.
- Stable release still requires external security review and production infrastructure integration.
- In this workspace, plain `pytest` can route localhost HTTP smoke-test traffic through system proxy software such as Privoxy. This is a local testing caveat, not a CAP protocol requirement. Use `source venv/bin/activate` followed by `NO_PROXY=127.0.0.1,localhost no_proxy=127.0.0.1,localhost pytest` for the documented baseline.
