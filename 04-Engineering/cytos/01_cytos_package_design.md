## 1. Purpose and posture

Cytos is the canonical Cytognosis package for:

**Data engineering layer**

1. authoring LinkML schemas across many biomedical domains (anatomy, disease, gene, variant, drug, phenotype, expression, pathway, sensor/SOSA, publication, GA4GH, NWB);
2. fetching, parsing, and license-gating upstream sources (UMLS, SNOMED CT, OLS4 SSSOM bundles, Biolink Model, BioThings APIs, GA4GH schemas, CELLxGENE schema versions, HDMF/NWB, Open Targets Parquet, OpenAlex, BibTeX, schema.org/Bioschemas, SOSA/SSN), plus the Cytognosis master dataset cohort (UKB, AoU, MVP, HPP, MGBB, ABCD, CCF/HCP, AMP-AD/ROSMAP/PPMI/ADNI/AMP-ALS/AMP-PDRD, PsychAD, ABIDE, B-SNIP, PARDIP);
3. harmonizing cross-source identifiers via OAK + SSSOM + optional SapBERT/LLM tiebreakers, with provenance sidecars in SSSOM format;
4. assembling a knowledge graph using BioCypher (direct-to-Neo4j, polymorphic adapters, head and tail ontology grafts) and Koza (Monarch-style source-to-KGX transforms), with cross-framework KGX merge;
5. publishing versioned snapshots (LinkML + Pydantic + JSON Schema + OWL + SHACL + KGX + Neo4j dump + DDE-JSON-LD + Schemasheets + PROV-O) under permissive license, with controlled subsets (UMLS/SNOMED-derived material, controlled-access biobanks) quarantined.

**Modeling layer (NEW; master-package scope)**

6. per-modality feature engineering and tokenization across the five Cytoverse modalities (phenotype, genotype, single-cell omics, connectomics/imaging, EHR);
7. multimodal co-embedding architectures: PoE/MoE/MoPoE VAE, contrastive cross-modal, Fused-Gromov-Wasserstein (FGW) barycenter, MOFA / MOFA-FLEX matrix factorization, cross-attention fusion transformers, Multiway/LIMoE/Soft mixture-of-experts, GNNs, scvi-tools foundation models, archetype analysis (MIDAA), and causal representation learning;
8. training infrastructure: Lightning trainers, JAX/Flax/Optax loops, MLflow experiment tracking, distributed DDP/FSDP/pmap, hyperparameter search, resumable checkpointing, ROCm/CUDA/CPU portable;
9. evaluation: integration metrics (scib), pairing-preservation metrics (CKA, RV, dCor, Procrustes), downstream-task probing, robustness benchmarks, ancestry/age/sex stratified reporting;
10. inference services: patient-embedding endpoint, batch inference for cohort scoring, KG-grounded Cypher generator, graph-RAG retrieval;

**LLM and RAG layer (optional extras)**

11. structured extraction (OntoGPT LinkML templates, Instructor + Pydantic AI, scispaCy hybrid), graph-RAG, LangGraph routers, grounded-answer enforcement.

Posture: **schema-and-model-first**. Every entity in cytos is a LinkML class; every external source is described by a typed record; every model architecture lives in a versioned module with a model card; every transformation, training run, and inference call ships a SHA-256 + license + ancestry/age/sex provenance header.

## 2. Naming and registry placement

Per Cytognosis four-tier naming register, `cytos` is the **foundation kernel** that absorbs the older `cytotype` placeholder. It sits one level below `cytognosis` (mission/org) and one level above neuro-scale repos (`neurogeno`, `neurocyto`, `neurotopo`, `neurophysio`, `neuroetho`). ## 4. Module-by-module functional summary

| Module | Stage | Functional summary |
|---|---|---|
| `cytos.schema` | linkmlize, publish | Author LinkML, regenerate every codegen target, run `gen-erdiagram` / `gen-mermaid-class-diagram` / `linkml-renderer` / `schemasheets` round-trip. Houses the format-bridge subpackage (JSON-LD to TTL, YAML JSON Schema to JSON, Protobuf to JSON Schema, Parquet schema to LinkML, RF2 to OWL, HDMF NamespaceCatalog walker) and the thin wrappers around `schemauto import-rdfs/-owl/-jsonschema` and `schemauto generalize-*`. |
| `cytos.sources` | download | Typed `SourceDescriptor` registry loaded from `configs/sources/*.yaml`. License-gated cache with renewal-date tracking (UMLS, SNOMED). Polite rate-limited HTTP/FTP. Per-source modules wrapping each upstream API (BioThings, OLS4, OT FTP, Monarch, OpenAlex, GA4GH, NWB). |
| `cytos.ingest` | parse, linkmlize | Per-source parser plus `linkmlize` driver that uses `schema.automator` + `schema.bridges` to emit LinkML-conformant artifacts. One submodule per playbook chapter source family. |
| `cytos.validate` | validate | Drives `linkml-validate`, `sssom validate`, `cellxgene-schema`, `nwbinspector`, `kgx validate`, generated SHACL, and referential integrity checks. Aggregates one JSON report per stage. |

## 5. Pipeline DAG and stage IO contracts

Stages, in dependency order, with the canonical IO contract for each:

**Data engineering arc**

1. **download** (`cytos.sources`)
   * Input: `configs/sources/<id>.yaml` (typed `SourceDescriptor`).
   * Output: `data/raw/<source>/<version>/<files...>` plus `data/raw/<source>/<version>/_manifest.json` with SHA-256, retrieval timestamp, license class.
2. **parse** (`cytos.ingest.parsers`)
   * Input: `data/raw/<source>/<version>/`.
   * Output: `data/interim/<source>/<version>/` (typed Parquet/JSONL or rdflib graph dumps).
3. **linkmlize** (`cytos.ingest.linkmlize`, `cytos.schema.automator`, `cytos.schema.bridges`)
   * Input: `data/interim/...`.
   * Output: (a) `schemas/domains/<topic>.yaml` (or `schemas/auto/<source>.yaml`) when the source contributes a new LinkML class; (b) `data/normalized/<source>/<version>/` Parquet keyed by LinkML target_class; (c) `data/linkml/<source>/<version>/` YAML/JSON serializations for validators.
4. **validate** (`cytos.validate`)
   * Input: `data/normalized/<source>/<version>/` + relevant `schemas/*.yaml`.
   * Output: `results/validation/<source>/<version>.json` plus exit-code summary; CI gate refuses to advance on errors.
5. **harmonize** (`cytos.harmonize`)
   * Input: normalized tables from multiple sources.
   * Output: `schemas/mappings/sssom/<pair>.sssom.tsv`, `data/normalized/<source>/<version>/_harmonized/`, `schemas/mappings/sssom/_provenance/<pair>.sssom.tsv`.
6. **kg_build** (`cytos.kg`)
   * Input: harmonized normalized tables.
   * Output: `kg/working/<release>/{nodes.tsv,edges.tsv}` then per-backend snapshots (neo4j-admin csvs, RDF, Parquet, DuckDB).
7. **qc_kg** (`cytos.validate.referential` + `cytos.validate.shacl` + `cytos.validate.kgx`)
   * Output: `results/coverage/<release>.json`, `results/validation/kg-<release>.json`.

**Modeling arc (NEW)**

8. **featurize** (`cytos.features`)
   * Input: harmonized normalized tables + cohort definition `configs/cohorts/<cohort>.yaml`.
   * Output: `data/cohorts/<cohort>/<version>/master.parquet` (patient-indexed join), `data/features/<cohort>/<version>/<modality>/` (Parquet/AnnData/Zarr per modality), `data/splits/<cohort>/<split>.yaml` (ancestry-stratified train/val/test).
9. **train** (`cytos.train`)
   * Input: `data/features/<cohort>/<version>/`, training config under `configs/training/<recipe>.yaml`.
   * Output: `checkpoints/<run>/`, MLflow run id, `results/training/<run>.json`, optional `models/pretrained/<arch>/<version>/` on graduate.
10. **evaluate** (`cytos.evaluate`)
    * Input: a trained model snapshot + held-out splits + benchmark fixtures under `benchmarks/datasets/`.
    * Output: `results/eval/<run>.json` (integration, pairing, downstream, fairness, robustness), updated `benchmarks/leaderboard.md`.
11. **kg_embed** (`cytos.models.kg_embed`)
    * Input: published KG snapshot + chosen entity subset.
    * Output: `data/embeddings/<run>/kg/` (entity + relation embeddings), saved as Parquet + checkpoint.
12. **infer** (`cytos.models.serve.batch`)
    * Input: a trained model + a cohort or single-patient bundle.
    * Output: `data/embeddings/<run>/<modality>/` per-patient embeddings, or `results/inference/<run>/<batch>.parquet`.

**Publication and serving**

13. **publish** (`cytos.publish`)
    * Input: validated KG + current `schemas/` + selected trained `models/pretrained/` + their cards + reference embeddings.
    * Output: `kg/snapshots/<release>/...` bundle (now also includes `models/`, `cards/`, `embeddings/reference/`) plus `locked/LOCK_MANIFEST.json` updated; optional Zenodo DOI + Hugging Face upload.
14. **serve_rag** (`cytos.rag`, optional)
    * Input: published snapshot.
    * Output: Cypher-generation, structured extraction, or graph-RAG service.

The full DAG (Mermaid):


## 9. Automated processing plan

A canonical "from zero to release" run:

```bash
# 0. Bootstrap (once per checkout)
git clone https://github.com/cytognosis/cytos.git
cd cytos && uv sync && nox -s setup_venv

# 1. Pre-flight: confirm every external dependency is reachable + licenses recorded
cytos sources doctor

# 2. Fetch every open source (controlled sources skipped unless UMLS_API_KEY etc. are set)
cytos sources fetch --all --license-class open

# 3. Parse + linkmlize all fetched sources
cytos ingest parse --all
cytos ingest linkmlize --all

# 4. Generate every codegen target from current schemas/
cytos schema generate --all

# 5. Validate every normalized table against its LinkML class
cytos validate linkml --all

# 6. Harmonize cross-source identifiers (writes SSSOM sidecars and rewrites normalized tables)
cytos harmonize resolve --all

# 7. Build the KG (BioCypher + Koza per the per-source decision registry)
cytos kg build cytos-2026.05

# 8. QC the KG (referential, SHACL, KGX)
cytos validate referential kg/working/cytos-2026.05/
cytos validate shacl   kg/working/cytos-2026.05/
cytos validate kgx     kg/working/cytos-2026.05/

# 9. Publish the snapshot (lock manifest, PROV-O, neo4j-admin csvs, DDE-JSON-LD)
cytos publish snapshot cytos-2026.05

# 10. (modeling arc) Build a cohort, featurize, train, evaluate, infer
cytos features build neuro-pilot                       # joins modalities for the neuro-pilot cohort
cytos features split neuro-pilot --stratify ancestry,sex,age
cytos train run configs/training/poe_vae_pilot.yaml     # Lightning + MLflow + ROCm
cytos evaluate integration <run-id>
cytos evaluate fairness   <run-id>
cytos kg-embed train kg/snapshots/cytos-2026.05/        # KG embedding via PyG
cytos infer batch <run-id> neuro-pilot                   # cohort-scale embeddings

# 11. (release) Bundle data + KG + models + cards + embeddings
cytos publish snapshot cytos-2026.05 --include-models pretrained/poe_vae@2026.05
cytos publish hf-upload poe_vae@2026.05                 # optional Hugging Face push
cytos publish zenodo cytos-2026.05                      # optional DOI mint
```

Each of these is also a nox session (`nox -s download`, `nox -s parse`, ..., `nox -s train`, `nox -s evaluate`, `nox -s infer`) and a Prefect flow under `pipelines/`. A weekly data build is encoded in `configs/pipelines/weekly_kg_build.yaml`; a monthly pretraining is encoded in `configs/pipelines/monthly_pretrain.yaml`. Both are surfaced via GitHub Actions cron in `.github/workflows/kg-build.yaml` and `.github/workflows/pretrain.yaml` (with manual approval for the pretrain workflow since it spends GPU time).

## 10. Provenance, license, and policy

Every artifact at every stage carries a sidecar header:

```yaml
source: <id>
source_version: <semver-or-date>
retrieval_date: <iso-8601>
license_class: open | open-restricted | controlled
license_spdx: <spdx-id>
sha256: <hex>
transform_commit: <git-sha>
agent: cytos@<version>
```

`policy/source_licenses.yaml` maps every source id to its license class. `policy/rego/gate.rego` rejects merges that promote `controlled` rows into the public snapshot. `policy/rego/kg_publish_gate.rego` enforces SHACL pass + zero controlled rows in the released bundle. PR-verify (`nox -s pr_verify`) collects these gates and fails the workflow if any are violated.

PROV-O serialization (`cytos.publish.prov`) writes an RDF graph alongside the main KG so every node and edge has retrievable provenance.

## 13. Open questions and follow-ups

1. Should `cytos.rag` ship in the core distribution or stay an extras-only add-on? Recommendation: extras-only; many cytos consumers will not use it.
2. SNOMED CT requires an Affiliate License plus a JRE; should cytos default to skipping it, or surface a one-line opt-in? Recommendation: skip by default, opt-in via `CYTOS_LICENSE_SNOMED=accepted` env var.
3. Should cytos vendor a tiny reference KG snapshot (e.g., 10k nodes built from open sources only) for tests and tutorials? Recommendation: yes, under `tests/data/reference_kg/`, max 50 MB, regenerated quarterly.
4. Multi-language schemas: cytos is python-only by profile, but LinkML is language-agnostic. Should we emit Go/Rust/TS Pydantic-equivalents under `artifacts/`? Recommendation: not in v1; revisit when a downstream consumer asks.
