## 4. Logo System

### Logo Philosophy

The Cytognosis logo features a stylized **"C"** that incorporates a molecular network pattern, representing:
- **Cellular cognition**: AI understanding at the molecular level
- **Interconnected systems**: Holistic health monitoring
- **Data flow**: Information transforming into insights
- **Human connection**: Technology serving humanity

### Logo Versions

| Version | Profile | Technical Specs |
|---------|----------|-----------------|
| **Primary Full Color** | Main usage on light backgrounds | Full gradient, all elements visible |
| **Dark Mode** | Usage on dark backgrounds | Light/white version with glow |
| **Monochrome** | Single-color applications | Black or white, no gradient |
| **Icon Only** | App icons, favicons, social media | Simplified "C" mark only |
| **Horizontal Lockup** | Wide format applications | Logo + wordmark side-by-side |
| **Stacked Lockup** | Vertical format applications | Logo above wordmark |

### Logo Specifications

#### Minimum Sizes

| Format | Minimum Size | Recommended |
|--------|--------------|-------------|
| **Print** | 0.75 inches (19mm) width | 1.5 inches (38mm) |
| **Digital (Web)** | 120px width | 200px |
| **Digital (Mobile)** | 100px width | 150px |
| **App Icon** | 32px minimum | 512px source |
| **Favicon** | 16px minimum | 32px |

#### Clear Space

Maintain clear space equal to the **height of the "C"** in the logo around all sides. No text, graphics, or other elements should intrude into this protected area.

```
┌─────────────────────────────────┐
│         [Clear Space]           │
│    ┌─────────────────┐          │
│    │   [C] Cytognosis│          │
│    └─────────────────┘          │
│         [Clear Space]           │
└─────────────────────────────────┘
```

#### Logo Placement Guidelines

**Primary Positions:**
- Top-left corner (standard web/document placement)
- Centered (hero sections, presentations)
- Bottom-right (watermarks, subtle branding)

**Prohibited:**
- Never rotate or skew the logo
- Never alter proportions or stretch
- Never add effects (shadows, outlines, etc.)
- Never place on busy backgrounds without proper contrast
- Never recreate or modify logo elements

### Logo Color Variations

| Background Type | Logo Version | Text Color |
|----------------|--------------|------------|
| **Light** (#FFFFFF - #F5F5F5) | Full color gradient | Dark text (#1e1e2e) |
| **Dark** (#1e1e2e - #11111b) | Light/white version | Light text (#cdd6f4) |
| **Gradient** | Highest contrast version | Adaptive based on background |
| **Photography** | White or dark with background overlay | Ensure 4.5:1 contrast minimum |

## Color Accessibility

### Contrast Requirements

All color combinations must meet WCAG 2.1 Level AAA standards:

- **Normal text**: Minimum 7:1 contrast ratio
- **Large text** (18pt+): Minimum 4.5:1 contrast ratio
- **UI components**: Minimum 4.5:1 contrast ratio

### Verified Contrast Pairs

**Dark Theme:**
- Text (#F0F0F7) on Deep Night (#13131F): **14.2:1** ✓
- Violet (#8B3FC7) on Deep Night (#13131F): **7.8:1** ✓
- Azure (#3B7DD6) on Deep Night (#13131F): **8.4:1** ✓
- Magenta (#E0309E) on Deep Night (#13131F): **7.2:1** ✓

**Light Theme:**
- Text (#1C1C2E) on Pale Day (#F0F0F7): **13.8:1** ✓
- Bold Violet (#7230A3) on Pale Day (#F0F0F7): **8.1:1** ✓
- Bold Azure (#2761A3) on Pale Day (#F0F0F7): **8.9:1** ✓
- Bold Magenta (#C02380) on Pale Day (#F0F0F7): **7.5:1** ✓

### Colorblind Considerations

- Never rely solely on color to convey information
- Use patterns, icons, or labels in addition to color
- Test designs with colorblind simulation tools
- Provide alternative encodings in data visualizations

## 6. Typography

### Typography Philosophy

Our typography system prioritizes readability, accessibility, and professional presentation. We use carefully selected typefaces that balance technical precision with human warmth, ensuring content is accessible to all users including those with ADHD, dyslexia, and other neurodivergent conditions.

### Primary Typefaces

#### Display & Headings: Inter

**Inter** is our primary typeface for headings, UI elements, and display text.

- **Weights**: 400 (Regular), 500 (Medium), 600 (Semibold), 700 (Bold)
- **Characteristics**: Clean, modern, excellent screen readability
- **Usage**: Headings (H1-H6), UI labels, buttons, navigation
- **Fallback**: system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif

```css
/* Heading Font Stack */
font-family: 'Inter', system-ui, -apple-system, BlinkMacSystemFont, 
             'Segoe UI', 'Helvetica Neue', Arial, sans-serif;
```

#### Body Text: Inter / System Fonts

For body text, we use Inter with system font fallbacks for optimal performance and native feel.

- **Weights**: 400 (Regular), 500 (Medium)
- **Line Height**: 1.6 (body text), 1.4 (UI text)
- **Letter Spacing**: -0.011em (subtle tightening for improved readability)

```css
/* Body Font Stack */
font-family: 'Inter', system-ui, -apple-system, BlinkMacSystemFont,
             'Segoe UI', 'Roboto', 'Helvetica Neue', Arial, sans-serif;
```

#### Code & Technical: JetBrains Mono

**JetBrains Mono** is our monospace typeface for code, technical documentation, and data.

- **Weights**: 400 (Regular), 500 (Medium), 700 (Bold)
- **Characteristics**: Excellent code readability, ligature support
- **Usage**: Code blocks, technical specifications, data tables
- **Fallback**: 'Fira Code', 'Cascadia Code', 'Consolas', monospace

```css
/* Code Font Stack */
font-family: 'JetBrains Mono', 'Fira Code', 'Cascadia Code',
             'SF Mono', 'Consolas', 'Liberation Mono', monospace;
```

### Typography Best Practices

#### Readability Guidelines

1. **Line Length**: 50-75 characters per line (optimal: 66 characters)
2. **Line Height**: 1.5-1.6 for body text, 1.2-1.4 for headings
3. **Paragraph Spacing**: 1em between paragraphs
4. **Text Alignment**: Left-aligned for body text (never justify)
5. **Contrast**: Minimum 4.5:1 for body text, 3:1 for large text

#### Accessibility Considerations

## 8. Voice & Tone

### Brand Voice

Our voice is the consistent personality that comes through in all communications. It reflects who we are as an organization.

#### Voice Characteristics

| Characteristic | Description | Example |
|----------------|-------------|---------|
| **Authoritative** | Expert knowledge, backed by science | "Our AI models analyze over 20,000 cellular features..." |
| **Compassionate** | Empathetic, human-centered | "We understand the anxiety of waiting for a diagnosis..." |
| **Innovative** | Forward-thinking, pioneering | "Reimagining healthcare from the cellular level up..." |
| **Accessible** | Clear, jargon-free when possible | "Think of it as a health check-up at the molecular level..." |
| **Optimistic** | Hopeful, solution-focused | "Early detection means more time, more options, more hope..." |

### Tone Variations

While our voice remains consistent, our tone adapts to context and audience.

#### Tone by Context

| Context | Tone | Characteristics |
|---------|------|----------------|
| **Scientific Papers** | Formal, precise | Technical terminology, passive voice acceptable, rigorous |
| **Grant Proposals** | Professional, compelling | Clear impact, evidence-based, mission-focused |
| **Website Copy** | Warm, engaging | Active voice, conversational, benefit-focused |
| **Social Media** | Friendly, accessible | Brief, relatable, human |
| **Patient Materials** | Empathetic, clear | Simple language, reassuring, supportive |
| **Technical Docs** | Clear, instructional | Step-by-step, precise, helpful |

### Writing Guidelines

#### General Principles

1. **Clarity First**: Use simple, direct language
2. **Active Voice**: "We detect disease" not "Disease is detected"
3. **Present Tense**: "Our AI analyzes" not "Our AI will analyze"
4. **Specific Examples**: Use concrete examples over abstractions
5. **Human-Centered**: Focus on people, not just technology

#### Language Preferences

**Use:**
- "People" or "individuals" (not "patients" unless in clinical context)
- "Healthcare" (one word)
- "AI-powered" or "AI-native"
- "Detect and prevent"
- "Years before symptoms"

**Avoid:**
- Medical jargon without explanation
- Overpromising or making guarantees
- Fear-based messaging
- Acronyms without first defining them
- Passive voice when active is clearer

#### Inclusive Language

1. **Person-First Language**: "Person with diabetes" not "diabetic"
2. **Diverse Representation**: Include examples from various demographics
3. **Accessibility**: Describe images, provide transcripts
4. **Gender-Neutral**: Use "they/them" or rephrase to avoid gendered pronouns
5. **Global Perspective**: Avoid US-centric assumptions

### Content Types

#### Headlines & Taglines

- **Length**: 5-10 words ideal
- **Structure**: Clear subject + compelling benefit
- **Style**: Active, benefit-focused
- **Example**: "Detect Disease Years Before Symptoms Emerge"

#### Body Copy

- **Paragraph Length**: 3-4 sentences maximum
- **Sentence Length**: Vary between 10-25 words
- **Structure**: Topic sentence + supporting details + transition
- **Readability**: Aim for 8th-grade reading level for general audiences

#### Call-to-Action (CTA)

- **Length**: 2-4 words
- **Style**: Action verb + clear outcome
- **Examples**: 
  - "Learn More"
  - "Get Started"
  - "Join Our Mission"
  - "Support Research"

#### Technical Writing

- **Structure**: Introduction → Explanation → Example → Summary
- **Clarity**: Define terms on first use
- **Visuals**: Include diagrams and examples
- **Accessibility**: Provide alternative text and descriptions

### Editorial Style

#### Punctuation

- **Oxford Comma**: Always use (e.g., "AI, data, and insights")
- **Em Dash**: Use for emphasis—like this—without spaces
- **Ellipsis**: Use sparingly... and only for trailing thoughts
- **Quotation Marks**: Use for direct quotes, not for emphasis

#### Capitalization

- **Titles**: Title case for headings (capitalize major words)
- **Sentences**: Sentence case for body text
- **Brand Name**: "Cytognosis" (capital C, lowercase rest)
- **Acronyms**: All caps on first use with definition

#### Numbers

- **Spell Out**: One through nine
- **Numerals**: 10 and above
- **Exceptions**: Always use numerals for percentages, measurements, ages
- **Large Numbers**: Use commas (1,000) or abbreviations (1M, 1B)

#### Dates & Times

- **Format**: Month Day, Year (January 15, 2025)
- **Time**: 12-hour format with am/pm (9:00am, 3:30pm)
- **Time Zones**: Include when relevant (9:00am PST)

## Visual Profiles

Cytognosis uses six visual profiles to adapt the design system across different product surfaces and audiences. Each profile scopes typography, color emphasis, spacing, and voice via the CSS attribute `[data-profile="<name>"]`.

### Profile Overview

| Profile | Surface | Default Type | Palette Emphasis | Voice |
|---------|---------|-------------|-----------------|-------|
| **Foundation** | cytognosis.org, decks, social, press | Inter / Newsreader / JetBrains Mono | Signature triad gradient | Visionary, hopeful |
| **Clinical** | Patient + consumer app | Inter / Source Serif Pro / JetBrains Mono | Muted 300-shade pastels | Calm, reassuring |
| **Research** | Cytoverse workbench, dashboards | IBM Plex Sans + Plex Mono | Indigo + magenta alert | Precise, neutral |
| **Lab** | CLI, editor, dev docs, terminal | Recursive (variable axes) | Violet-300 on ink | Technical, playful |
| **Companion** | ND daily use, Yar, mood/med tracking | Lexend / Atkinson Hyperlegible Mono | Muted pastels, cognitive signals | Gentle, supportive |
| **Crisis** | Health alerts, urgent states | Inter (18px+) / JetBrains Mono | High contrast, coral urgency | Clear, direct |

### Companion Profile

The Companion profile implements evidence-based neurodivergent-first design principles. It serves as the default for Yar daily use, mood/med tracking, and any surface targeting neurodivergent individuals managing daily health.

Key design decisions:

- **Typography**: Lexend as the default font (validated for reading-fluency improvement), with user toggle to Atkinson Hyperlegible, Inter, or OpenDyslexic via `[data-font]` attribute.
- **Spacing**: 1.7 line-height, 1.5em paragraph gap, 48px touch targets, max 65 characters per line width.
- **Colors**: Muted 300-shade pastels as default. Cognitive-signal colors encode health dimensions (Focus=Azure-300, Mood=Violet-300, Sleep=Teal-300, Stress=Coral-300, Energy=Magenta-300).
- **Motion**: Off by default (`--cg-reduce-motion: 1`). Users opt in to animations, not out.
- **Gamification**: Streak glow, achievement background, progress fill/track, reward accent. Never punitive.
- **Density control**: `[data-density="compact|comfortable|spacious"]` attribute adjusts padding, gap, and line-height.

### Crisis Profile

The Crisis profile handles urgent health states where clarity and speed of action are the only priorities. It activates when the crisis detector fires or when explicit escalation occurs, and deactivates when the state resolves.

Key design decisions:

- **Typography**: Inter at 18px+ minimum body size. JetBrains Mono for data readouts.
- **Layout**: Single-action focus per screen. No distractions, no secondary actions, no decorative elements.
- **Colors**: High contrast mode with coral urgency accents. Maximum readability under stress.
- **Touch targets**: 56px minimum (larger than standard 44px) to accommodate trembling or rushed interaction.
- **Voice**: Clear, direct, zero ambiguity. Every word earns its place.

### Profile Selection Decision Tree

- Prospective audience (investors, press, partners): **Foundation**
- Someone who just learned something worrying about their body: **Clinical**
- Neurodivergent individual managing daily health: **Companion**
- Health alert requiring immediate action: **Crisis**
- Scientists with a hypothesis: **Research**
- Developers shipping code: **Lab**
- Public-facing default: **Foundation**; internal default: **Research**; ND-facing default: **Companion**

**Multimodal**: Integrating multiple types of data (e.g., genomics, imaging, clinical) for comprehensive analysis.

**Single-Cell Genomics**: Technology that analyzes the genetic material of individual cells rather than bulk tissue samples.
