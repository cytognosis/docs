## 3. Comparative Landscape Analysis (Updated)

The neurodivergent productivity tool market splits into six distinct clusters. V3 adds voice-first capture tools (OMI, Letterly) as a new cluster and incorporates design research (Blue Lin) as evaluation criteria.

### Cluster F: Voice-First Capture Tools (NEW)
**OMI AI** and **Letterly**

These tools validate that voice-first interaction eliminates the "activation energy" barrier that is the defining friction point for ADHD users. OMI captures continuously via dedicated hardware (pendant, $89); Letterly records on-demand via phone microphone. Both apply AI to structure raw speech into usable output. OMI's MIT-licensed full stack (12.6k GitHub stars) provides the most comprehensive open-source reference for building a voice-first AI companion. Letterly's 25+ rewrite styles and $200K+ MRR (bootstrapped) prove commercial viability of the "speak messy, get clean" workflow.

The critical gap in both tools: they capture beautifully but do nothing with the captured information. OMI extracts action items as static text (no task tracking). Letterly generates formatted output (no task, no knowledge graph). Neither tool helps users execute, manage time, or regulate emotions. Yar closes this gap by connecting the capture pipeline to task management, knowledge graph, focus mode, and emotional awareness.

### Cluster A: Task-Focused Execution Engines
**Leantime** and **Super Productivity**

Both tools center on getting tasks done. Super Productivity (86/120) leads the entire comparison in raw feature coverage with Pomodoro timers, idle detection, focus mode, and keyboard-first interface. Codebase analysis reveals a `FocusModeStrategy` pattern supporting three distinct focus strategies (Pomodoro, flowtime, countdown), making it the most mature implementation of ND-specific time management. Leantime (81/120) targets teams with Kanban boards, Gantt charts, and sprint management. Its hexagonal architecture with 57 domain modules demonstrates how to build an enterprise-grade PM tool that respects ADHD users through emoji mood tracking and LEO AI.

### Cluster B: Knowledge-Focused Organizers
**Tana**, **Capacities**, and **Anytype**

These tools prioritize capturing, connecting, and retrieving knowledge. Tana's Supertag system represents the most sophisticated personal knowledge graph with schema inheritance, composition, and queries-as-first-class-objects. Capacities offers a cleaner "thinking in things" UX. Anytype combines local-first data sovereignty with E2E encryption, making it Yar's planned storage backend. Blue Lin's research validates that the most effective visualization of longitudinal personal data uses progressive disclosure (summary → detail → raw data → comparison), supporting multiple mental models (phase-first vs. signal-first).

### Cluster C: AI-Augmented Assistants
**Goblin Tools** and **Saner AI**

AI drives the core interaction model. Goblin Tools provides focused micro-utilities that each solve one executive function challenge. The spiciness slider for task decomposition granularity is a genuinely novel UX invention validated by the ADHD paper (Chen et al. 2026) as addressing task initiation paralysis. Saner AI's Skai agent proactively extracts tasks from email, calendar, and notes. Chen et al. validate proactive morning check-ins (Concept 2) as beneficial when combined with mood awareness.

### Cluster D: Accessibility-First Utilities
**Speechify**

Speechify transforms reading into listening, creating dual-channel input for dyslexia and attention difficulties. Letterly is its conceptual opposite: Speechify converts text → speech, Letterly converts speech → text. Together, they form the complete bidirectional voice pipeline that Yar aims to integrate.

### Cluster E: Composable Infrastructure
**ND Visual Organizer (MCP)** and **Anytype**

Protocol-first, maximum data sovereignty, but requiring technical sophistication. OMI's MCP server implementation provides a production reference for exposing cognitive data to external AI assistants.

## 7. Blue Lin Design Principles Applied to Yar

Blue Lin's research (Columbia University, PhD Toronto) surfaces five design principles for health data visualization that transfer directly to cognitive state tracking:

| # | Principle | Blue Lin Application | Yar Application |
|:---|:---|:---|:---|
| 1 | **Sensemaking over prediction** | Help users understand cycle patterns, not just predict dates | Help users understand cognitive patterns, not just schedule tasks |
| 2 | **User agency as primary goal** | Users control what signals to view and how | Yar illuminates patterns; users decide what to do |
| 3 | **Multimodal integration without overwhelm** | Progressive disclosure: summary → detail → raw data | Default composite "How am I doing?" card → signal groups → raw data |
| 4 | **Longitudinal context is essential** | Multi-cycle overlay reveals patterns single-day views miss | Multi-week overlay reveals cognitive signature |
| 5 | **Inclusive, culturally responsive design** | Gender-inclusive menstrual tracking | Neurotype-neutral, culturally responsive cognitive tracking |

### Five Functional Design Requirements (Lin CHI 2024 → Yar)

| DR# | Lin Requirement | Yar Equivalent |
|:---|:---|:---|
| DR1 | Communicate predicted phases + variance | Communicate predicted cognitive states with confidence ranges |
| DR2 | Support different interaction patterns | Daily check-ins, weekly summaries, on-demand deep-dives, crisis-only modes |
| DR3 | Personalize viewable signals | User chooses which cognitive signals to track and display |
| DR4 | Integrate educational resources | Explain neuroscience behind observed patterns |
| DR5 | Side-by-side comparison with context | Compare weeks/months with contextual annotations |

### Visualization Technique Catalog

| Technique | Data Type | Yar Application |
|:---|:---|:---|
| Phase-coded timeline | Temporal + categorical | Color-code daily timeline by cognitive state (focus/transition/rest/recovery) |
| Multi-cycle overlay | Longitudinal temporal | Overlay weekly focus scores aligned by day-of-week |
| Progressive disclosure dashboard | Summary → detail | Composite state card → signal groups → raw data |
| Signal heatmap | Multivariate temporal | Week-by-hour heatmap of focus/energy levels |
| Contextual annotation overlay | Event + temporal | Life event markers visible across all views |
| Empathetic data framing | Tone/language | "Your focus was lower today" not "You had a bad day" |

## 11. Design Tensions and Trade-offs

The ADHD paper and 12-tool analysis surface four fundamental tensions Yar must navigate:

| Tension | Paper Finding | Tool Evidence | Yar Response |
|:---|:---|:---|:---|
| **Capture vs. Execution** | Users need both ambient capture AND structured task management | OMI captures beautifully (9/10 AI) but has no task management (3/10). Super Productivity executes beautifully (9/10 tasks) but has no voice capture. | Yar unifies both: voice capture → AI extraction → persistent task objects |
| **Autonomy vs. Guidance** | "I want it to suggest, not decide." (S14) | Saner AI proactively plans; Goblin Tools is passive/on-demand. Users prefer different autonomy levels. | CAP enforces companion authority. All suggestions are inspectable, adjustable, overridable. User controls autonomy level. |
| **Privacy vs. Intelligence** | Participants prefer AI that doesn't surveil (Section 7.3) | OMI is cloud-first (Firebase + Pinecone). Super Productivity is fully local. | Yar is local-first by default. On-device Gemma inference. Optional cloud for enhanced quality. |
| **Depth vs. Breadth** | Each tool excels in one area but misses others | No tool covers >70% of validated features. Letterly (41) and Speechify (46) are deep but narrow. Super Productivity (86) is broad but misses voice/AI/emotion. | Yar targets 107/120 by unifying all six clusters |
| **Persona Consistency vs. Adaptability** | Users want a companion that "knows them" but also adapts to context | Tana has one voice. OMI has none. No tool offers mood-aware persona switching. | Yar learns multiple personas and switches based on mood-context model. Core persona identity persists; interaction style adapts. |
