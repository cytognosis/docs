# The Map of Everything, in Plain Words

**Reading time: 1 minute.**

> **If you only read one thing:** the docs repo's 8 numbered folders are THE categories, everywhere. Each of your project folders is a workspace for exactly one of them. Design now has its own project, like it has its own folder in docs.

| Your project folder | Is the workspace for | In one phrase |
|---|---|---|
| Grants | Funding | money in: grants, funders, applications |
| Science and Platform | Research | **the Neuroverse science home**: biotypes, psych axes, platform science |
| Design | Design | brand, design system, templates |
| Website | Products (website part) | the public site |
| Yar | Products (Yar part) | the ND app and its path to market |
| Cytos | Engineering (schemas part) | data standards and ontologies |
| Infrastructure and Tooling | Engineering | servers, CI, agent tools, the memory system, the task board |
| Operations | Operations | legal, comp, HR, filings |
| Cytognosis | Strategy | org-level strategy and structure |
| Refactor | Engineering (meta) | the consolidation cockpit |
| Personal | (private, outside the shared docs) | cv, biosketch, networking |

Rule of thumb: **drafts in the project, finished docs in the docs repo, data in the datasets folder.** If you ever wonder where something goes, open PROJECT-MAP.md in the docs repo; it is the contract.
